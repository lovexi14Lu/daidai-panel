## 修复

### Magisk 模块无法打开面板页面

- 修复 v2.0.6 面具版安装后浏览器访问 `http://127.0.0.1:5700` 打不开的问题。
- 原因：Docker 部署靠 nginx 在 5700 提供前端并反代 `/api` 到后端 5701，而 Magisk 模块里没有 nginx，只有 `daidai-server` 在 5701 上提供 API，5700 端口实际没有任何服务监听。
- 修复方案：`daidai-server` 新增 `server.web_dir` 配置项（也可用环境变量 `WEB_DIR`），配置后会直接托管前端静态目录，并对前端路由做 SPA fallback。Magisk 模块改为单端口 5700，由后端同时服务 API 和前端。
- 老用户升级：安装包里的 `customize.sh` 会自动把 `/data/adb/daidai-panel/config.yaml` 中的 `port: 5701` 改写为 `5700`，并补上 `web_dir: /data/adb/modules/daidai-panel/web`。数据目录不受影响。
- Docker / 宿主机部署不受影响：`web_dir` 未配置时保持原有行为（前端由 nginx 托管）。
